# AI 代理开发指南

## 1. 核心原则
- **中文回复**：始终使用简体中文，代码注释也用中文。
- **严谨求真**：绝不瞎猜。不确定时先查证或提问。区分事实与推测。
- **先读后改**：修改代码前必须先阅读相关文件和上下文。
- **最小修改**：只做实现目标所需的最小必要变更，不随意改动无关代码。

## 2. 架构规范 (关键)
- **模块化设计**：每个功能逻辑独立成文件（单一职责原则）。
- **路由调度**：由统一的入口/路由文件（如 `index.ts`）负责事件分发和调度。
- **禁止巨型文件**：严禁将所有逻辑堆砌在单个文件中。

## 3. 代码规范
- **格式**：2 空格缩进，分号结尾。
- **命名**：
  - 变量/函数：`camelCase` (如 `handleMessage`)
  - 类/接口：`PascalCase` (如 `SessionStore`)
  - 文件名：`kebab-case` (如 `user-session.ts`)
- **类型安全**：严禁使用 `any`，必须定义明确的 TypeScript 类型。

## 4. 工作流
- **计划先行**：复杂任务先制定步骤。
- **自我验证**：修改后必须考虑如何验证功能。
